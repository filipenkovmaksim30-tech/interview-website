Вопросы для Java Junior Backend разработчика

  1. В чём разница между JDK, JRE и JVM?

    JVM (Java Virtual Machine) — исполнительная среда, выполняющая байт-код. JRE (Java Runtime Environment) = JVM + библиотеки классов, минимально необходимое для запуска Java-программ. JDK (Java Development Kit) = JRE + инструменты разработчика (компилятор javac, отладчик, утилиты). Для разработки нужна JDK, для запуска — JRE (или JDK).

  2. Объясните принципы ООП: наследование, инкапсуляция, полиморфизм. Как они реализованы в Java?

    Наследование — создание нового класса на основе существующего (extends), перенимая его поля и методы. Инкапсуляция — сокрытие внутренней реализации через модификаторы доступа (private, protected, public) и предоставление доступа через геттеры/сеттеры. Полиморфизм — способность объекта вести себя по-разному в зависимости от типа: переопределение методов (runtime) и перегрузка (compile-time). В Java всё это поддерживается на уровне языка и JVM (виртуальные методы).

  3. Что такое ArrayList и чем он отличается от LinkedList? Когда что использовать?

    ArrayList основан на динамическом массиве, обеспечивает быстрый доступ по индексу O(1), но медленную вставку/удаление из середины O(n). LinkedList — двусвязный список, быстрая вставка/удаление в начале/середине O(1), но медленный доступ O(n). ArrayList предпочтителен для частого чтения, LinkedList — когда много вставок/удалений в середине списка.

  4. Что такое try-catch-finally и зачем нужен блок finally?

    Конструкция для обработки исключений. В try помещается код, который может выбросить исключение. catch перехватывает и обрабатывает конкретный тип исключения. finally выполняется всегда (кроме System.exit()), независимо от того, было исключение или нет. Используется для освобождения ресурсов (закрытие потоков, соединений).

  5. Объясните разницу между == и методом equals().

    Оператор == сравнивает примитивы по значению, а ссылочные типы — по адресу в памяти. Метод equals(), унаследованный от Object, по умолчанию тоже сравнивает ссылки, но обычно переопределяется для сравнения содержимого объектов (например, String, Integer). Правило: для примитивов — ==, для объектов по содержимому — equals().
