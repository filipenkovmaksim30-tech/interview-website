Вопросы для Java Middle Backend разработчика

  1. Как устроен HashMap в Java? Почему важно правильно реализовать hashCode() и equals()?

    HashMap основан на массиве бакетов (bucket), каждый из которых — связный список или красно-чёрное дерево (при большом количестве коллизий). При добавлении ключа вычисляется хеш (hashCode), по нему определяется индекс бакета. Затем внутри бакета ищется элемент с равным ключом через equals(). Если hashCode() реализован плохо (много коллизий), производительность падает до O(n). Обязателен контракт: если два объекта равны по equals(), их hashCode() обязаны быть одинаковыми.

  2. Что такое пул строк (String Pool) и почему строки в Java неизменяемые (immutable)?

    Пул строк — область памяти (часть heap), где хранятся уникальные строковые литералы. При создании строки через литерал JVM сначала ищет её в пуле, если находит — возвращает ссылку на существующую. Неизменяемость обеспечивает безопасность (строки можно переиспользовать без риска изменения), потокобезопасность и возможность кеширования хеша. Строки, созданные через new String(), в пул автоматически не попадают, но можно вызвать intern().

  3. Какие способы создания потоков есть в Java? Опишите жизненный цикл потока.

    Способы: наследование от класса Thread, реализация интерфейса Runnable, реализация Callable (возвращает результат через Future), использование пулов потоков ExecutorService. Жизненный цикл: NEW (создан, но не запущен) → RUNNABLE (работает или готов к выполнению) → BLOCKED (ждёт монитор) / WAITING (ожидает notify) / TIMED_WAITING (с таймаутом) → TERMINATED (завершён).

  4. Что такое Spring Boot и какие преимущества он даёт перед обычным Spring Framework?

    Spring Boot — расширение Spring, упрощающее создание самостоятельных приложений с минимумом конфигурации. Ключевые особенности: автоконфигурация (на основе зависимостей), встроенные серверы (Tomcat, Jetty), стартеры (объединённые зависимости), Actuator для мониторинга, отсутствие XML-конфигурации. Позволяет быстро развернуть микросервис с минимальным кодом.

  5. Зачем нужна сериализация в Java? Чем различаются интерфейсы Serializable и Externalizable?

    Сериализация — преобразование объекта в поток байтов для сохранения в файл, передачи по сети или кеширования. Serializable — маркерный интерфейс, сериализация выполняется автоматически с использованием рефлексии, можно управлять через transient и методы writeObject/readObject. Externalizable — полностью контролируемая сериализация, требует реализации методов writeExternal и readExternal, обычно быстрее, но код писать сложнее.
