Вопросы для C++ Middle Backend разработчика

  1. Что такое move-семантика и для чего она нужна? Объясните понятия lvalue и rvalue.

    Move-семантика позволяет перемещать ресурсы из одного объекта в другой без глубокого копирования, что повышает производительность. lvalue — выражение, у которого есть адрес в памяти (переменная). rvalue — временное выражение без постоянного адреса (результат функции, литерал). Конструктор перемещения и оператор перемещающего присваивания работают с rvalue-ссылками (T&&). Ключевая функция std::move() преобразует lvalue в rvalue-ссылку, явно показывая намерение переместить объект.

  2. Какие виды многопоточности доступны в C++11 и новее? Расскажите про std::thread, std::mutex, std::async и std::future.

    C++11 ввёл стандартную библиотеку для многопоточности. std::thread — низкоуровневый класс, запускающий функцию в отдельном потоке. std::mutex и его варианты (recursive_mutex, shared_mutex в C++17) обеспечивают синхронизацию, а std::lock_guard и std::unique_lock — RAII-обёртки для удобной и безопасной блокировки. std::async запускает асинхронную задачу и возвращает std::future — объект, из которого можно позже получить результат. std::promise позволяет вручную установить значение в связанный std::future. Это основа для построения конкурентных backend-систем.

  3. Что такое шаблоны (templates) и чем отличаются шаблоны функций от шаблонов классов? Что такое специализация шаблона?

    Шаблоны — механизм обобщённого программирования, позволяющий писать код, работающий с разными типами без дублирования. Шаблон функции выводит типы из аргументов вызова, шаблон класса требует явного указания типа при инстанцировании (например, std::vector). Специализация шаблона — отдельная реализация для конкретного типа. Полная специализация заменяет код для конкретного типа целиком, частичная — для подмножества типов (например, для всех указателей T*). Это важно для оптимизации и особого поведения для определённых типов.

  4. Что такое исключения (exceptions) в C++? Как работает stack unwinding и гарантии безопасности исключений?

    Исключения — механизм обработки ошибок, прерывающий нормальный поток выполнения при возникновении нештатной ситуации. При выбрасывании исключения происходит stack unwinding: выполнение поднимается вверх по стеку вызовов, уничтожая все локальные объекты (вызываются деструкторы), пока исключение не будет поймано в try/catch. Гарантии безопасности: базовая (нет утечек ресурсов, объекты в валидном состоянии), строгая (состояние программы не меняется при неудаче, как у транзакций), noexcept (операция не бросает исключений). Правильное понимание этих гарантий критично для надёжного backend-кода.

  5. Что такое виртуальные функции и как работает таблица виртуальных методов (vtable)? Зачем нужен виртуальный деструктор?

    Виртуальные функции — механизм полиморфизма, позволяющий вызывать метод производного класса через указатель/ссылку на базовый класс. Каждый класс с виртуальными функциями имеет скрытую таблицу виртуальных методов (vtable) — массив указателей на фактические реализации. Объект содержит скрытый указатель vptr на vtable своего класса. Вызов виртуальной функции идёт через поиск в vtable, что даёт небольшие накладные расходы. Виртуальный деструктор необходим, когда объект удаляется через указатель на базовый класс — без него деструктор производного класса не вызовется, что приведёт к утечке ресурсов и неопределённому поведению.
