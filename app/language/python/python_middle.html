Вопросы для Python Middle Backend разработчика

  1. Что такое ORM и какие преимущества даёт использование Django ORM / SQLAlchemy?

    ORM (Object-Relational Mapping) — прослойка между объектным кодом и реляционной базой данных. Позволяет работать с записями как с объектами Python, автоматически генерирует SQL, защищает от SQL-инъекций, упрощает миграции. В Django ORM тесно интегрирована с моделями и миграциями, SQLAlchemy даёт больше гибкости (может использоваться как Core, так и ORM).

  2. Что такое асинхронное программирование и как в Python реализованы async/await?

    Асинхронность позволяет не блокировать выполнение программы в ожидании I/O операций. Конструкции async/await и event loop (asyncio) переключают контекст между корутинами, не создавая новых потоков. Подходит для высоконагруженных сетевых сервисов. Важно: CPU-bound задачи асинхронность не ускоряет.

  3. Как работают генераторы и итераторы, и когда их использовать?

    Итератор — объект, реализующий метод __next__(). Генератор — функция с yield, автоматически создающая итератор. Генераторы ленивы: вычисляют значения по запросу, экономят память. Применяются при работе с большими потоками данных (файлы, стриминг), пагинация, бесконечные последовательности.

  4. Объясните разницу между @staticmethod, @classmethod и обычным методом класса.

    Обычный метод принимает self (экземпляр). @classmethod принимает cls (класс), может использоваться для фабричных методов. @staticmethod не принимает ни self, ни cls, по сути — обычная функция внутри класса. Выбор зависит от того, нужен ли доступ к атрибутам экземпляра или класса.

  5. Какие типы тестов вы знаете и зачем нужно тестирование? Чем юнит-тесты отличаются от интеграционных?

    Типы: модульные (юнит), интеграционные, функциональные (e2e). Юнит-тесты проверяют отдельные компоненты в изоляции (моки). Интеграционные — взаимодействие между компонентами (с реальной БД, внешними сервисами). Тестирование гарантирует работоспособность, упрощает рефакторинг, служит документацией.
