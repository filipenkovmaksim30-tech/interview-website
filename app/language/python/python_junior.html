Вопросы для Python Junior Backend разработчика

  1. Что такое декоратор в Python и для чего он используется?

    Декоратор — это функция, которая принимает другую функцию и расширяет или изменяет её поведение без изменения исходного кода. Часто применяется для логирования, проверки прав доступа, кэширования результатов и замера времени выполнения. Синтаксис: @decorator_name перед определением функции.

  2. Чем отличаются списки (list) от кортежей (tuple)?

    Список (list) — изменяемый тип данных: можно добавлять, удалять и изменять элементы. Кортеж (tuple) — неизменяемый. После создания его содержимое нельзя изменить. Благодаря неизменяемости кортежи могут использоваться как ключи в словарях, а списки — нет. Операции с кортежами обычно выполняются быстрее.

  3. Что такое GIL (Global Interpreter Lock) и как он влияет на многопоточность?

    GIL — это блокировка в интерпретаторе CPython, которая позволяет выполняться только одному потоку Python одновременно, даже на многоядерном процессоре. Из-за GIL многопоточные программы на Python неэффективны для вычислительных задач (CPU-bound), но отлично подходят для ввода-вывода (I/O-bound), так как во время ожидания ответа от сети или диска блокировка снимается.

  4. Какие структуры данных в Python вы знаете и когда какую использовать?

    Основные встроенные структуры: списки (list), кортежи (tuple), словари (dict), множества (set) и неизменяемые множества (frozenset). Списки — для упорядоченных изменяемых коллекций, кортежи — для фиксированных наборов, словари — для хранения пар ключ-значение с быстрым доступом, множества — для уникальных элементов и операций над множествами. Выбор зависит от требований к скорости доступа, необходимости изменять данные и наличия дубликатов.

  5. Объясните разницу между методами GET и POST в HTTP.

    GET-запрос используется для получения данных с сервера. Параметры передаются в URL, есть ограничения по длине, запросы могут кэшироваться и оставаться в истории браузера. POST-запрос применяется для отправки данных на сервер (например, форма с паролем). Данные передаются в теле запроса, не видны в URL, не кэшируются и не имеют строгих ограничений по размеру. Основное правило: GET — для чтения, POST — для изменений на сервере.
