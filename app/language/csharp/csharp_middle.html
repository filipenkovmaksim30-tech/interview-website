Вопросы для C# Middle Backend разработчика

  1. Как работает сборщик мусора в .NET? Опишите поколения и режимы работы.

    GC работает по поколениям (generational): Gen0 — новые объекты, Gen1 — пережившие одну сборку, Gen2 — долгоживущие. Сборка мусора происходит, когда Gen0 заполнен, выжившие продвигаются выше. Большие объекты (LOH, >85 КБ) размещаются отдельно и собираются реже. Режимы: workstation (для клиентских приложений, минимизация пауз), server (многопоточный, для серверной нагрузки). GC можно влиять через методы GC.Collect (не рекомендуется) и WeakReference.

  2. Что такое async/await и как он работает на уровне компилятора? Что происходит с потоком во время await?

    async/await — асинхронная модель для неблокирующего выполнения операций. Компилятор преобразует async-метод в конечный автомат, разделяя его на части до и после await. Во время await управление возвращается вызывающему коду, поток не блокируется, а освобождается для других задач. Когда асинхронная операция завершается, выполнение продолжается (обычно на том же потоке, если контекст синхронизации позволяет).

  3. Что такое LINQ и какие виды операторов вы знаете? В чём разница между IEnumerable и IQueryable?

    LINQ (Language Integrated Query) — способ запросов к данным прямо в C#. Операторы делятся на отложенные (where, select, take) и немедленные (Count, ToList). IEnumerable работает с данными в памяти, запрос выполняется по мере итерации. IQueryable используется для удалённых источников (БД), формирует дерево выражений, которое транслируется в SQL (или другой язык запроса) на стороне сервера.

  4. Как работает Dependency Injection в ASP.NET Core? Какие lifetime у сервисов существуют?

    DI — встроенный контейнер, который автоматически разрешает зависимости. Регистрация происходит в IServiceCollection. Жизненные циклы: Transient (новый экземпляр при каждом запросе), Scoped (один экземпляр на HTTP-запрос), Singleton (единый экземпляр на всё приложение). Правильный выбор lifetime критичен для производительности и корректности работы (например, не внедрять Scoped-сервис в Singleton).

  5. Что такое Entity Framework Core и какие подходы к работе с данными он поддерживает?

    EF Core — ORM для .NET. Поддерживает подходы: Code First (модель описывается классами, по которым создаётся БД), Database First (модель генерируется из существующей БД). Отслеживание изменений, миграции, LINQ-запросы трансформируются в SQL. Важно понимать проблему N+1, eager loading (Include), lazy loading и отслеживаемые/неотслеживаемые запросы.
