Вопросы для C# Senior Backend разработчика

  1. Объясните устройство Roslyn (компилятор как сервис). Как использовать анализаторы и генераторы кода?

    Roslyn — компилятор C# и VB.NET с открытым API. Позволяет анализировать код, строить синтаксические деревья, искать семантическую информацию. Анализаторы — расширения, проверяющие код на соответствие правилам (стиль, безопасность) и генерирующие предупреждения или исправления (code fix). Генераторы исходного кода (source generators) — генерируют код во время компиляции, уменьшая рефлексию (например, генерация сериализации/маппингов). Широко используются в современных библиотеках (System.Text.Json, Minimal APIs).

  2. Как устроен Kestrel и обработка HTTP-запроса в ASP.NET Core? Что такое Middleware pipeline?

    Kestrel — встроенный высокопроизводительный веб-сервер на основе libuv/Socket. Обработка: запрос попадает в Kestrel, передаётся в конвейер Middleware. Middleware — компоненты, обрабатывающие запрос по цепочке (логирование, авторизация, маршрутизация). Каждый Middleware вызывает следующий и может изменить ответ. Порядок регистрации важен. После Middleware запрос доходит до Endpoint (MVC контроллер или Minimal API).

  3. Какие механизмы для реализации фоновых задач в ASP.NET Core вы знаете? Сравните IHostedService, BackgroundService и очереди сообщений.

    IHostedService / BackgroundService — абстракции для запуска длительных фоновых операций в том же процессе. Подходят для лёгких периодических задач (очистка, синхронизация). Для масштабируемой фоновой обработки используют внешние очереди сообщений (RabbitMQ, Azure Service Bus) и воркеры (Worker Service). Это позволяет вынести нагрузку, обеспечить устойчивость к перезапускам и балансировку.

  4. Расскажите о стратегиях кэширования в высоконагруженных .NET приложениях. Как использовать In-Memory Cache, Distributed Cache и Output Cache?

    In-Memory Cache (IMemoryCache) — хранит данные в памяти процесса, быстрый, но не подходит для фермы серверов. Distributed Cache (IDistributedCache) — для shared кэша (Redis, SQL Server), медленнее, но синхронизирован между экземплярами. Output Cache — кэширует HTTP-ответы целиком (Response Caching). Важно учитывать инвалидацию, время жизни и стратегии обновления (cache-aside, refresh-ahead). Также полезны библиотеки третьих сторон (CacheManager, EasyCaching).

  5. Как обеспечить потокобезопасность и минимизировать блокировки в многопоточном серверном приложении на C#? Опишите lock, SemaphoreSlim, Channel.

    Потокобезопасность достигается синхронизацией доступа к разделяемым данным. lock (Monitor) — простой мьютекс, блокирует выполнение другими потоками. Может приводить к deadlock при неправильном порядке вложенных блокировок. SemaphoreSlim — ограничивает параллелизм (пул заданий). Channel (System.Threading.Channels) — асинхронная очередь с поддержкой BoundedCapacity и backpressure. Для снижения блокировок используют lock-free структуры (ConcurrentDictionary, Interlocked), стараются проектировать асинхронное взаимодействие через Task и dataflow.
