Вопросы для C# Junior Backend разработчика

  1. Что такое .NET и из каких компонентов он состоит? Чем отличается .NET Framework от .NET (ранее .NET Core)?

    .NET — это платформа для разработки и запуска приложений. Состоит из CLR (Common Language Runtime) — среды выполнения, и библиотек классов (FCL/BCL). .NET Framework — старая, Windows-only версия. .NET (/.NET Core) — современная, кроссплатформенная, с открытым исходным кодом, более производительная и модульная. Сейчас основной вектор развития — .NET 5/6/7/8+ (унифицированная платформа).

  2. Что такое value-типы и reference-типы? Где хранятся те и другие?

    Value-типы (int, float, struct) хранят значение непосредственно в переменной, размещаются в стеке (или внутри объекта в куче). При присваивании создаётся копия значения. Reference-типы (class, string, array) хранят ссылку на объект в куче. При присваивании копируется ссылка, объект остаётся один. Понимание разницы важно для производительности и избежания ошибок с null.

  3. Объясните ключевые слова ref, out и in при передаче параметров.

    ref — передача по ссылке, требует инициализированную переменную до вызова. Метод может изменить значение. out — передача по ссылке для выходных параметров, переменная может быть не инициализирована при вызове, метод обязан присвоить значение. in — передача по ссылке только для чтения, значение нельзя изменить внутри метода. Обычно используется для избежания копирования больших структур.

  4. Что такое интерфейс и зачем он нужен? Может ли класс реализовать несколько интерфейсов?

    Интерфейс (interface) определяет контракт — набор методов и свойств без реализации, который класс обязан реализовать. Позволяет достичь полиморфизма и слабой связанности (dependency injection). В C# класс может наследоваться только от одного класса, но реализовывать множество интерфейсов, что решает проблему отсутствия множественного наследования.

  5. Что такое исключения и как работает конструкция try-catch-finally?

    Исключения — способ обработки ошибок во время выполнения. В блоке try пишется код, который может вызвать ошибку. Если исключение возникает, управление передаётся в подходящий catch. Блок finally выполняется всегда (даже после return или unhandled exception), используется для очистки ресурсов (закрытие файлов, соединений). Также есть using — синтаксический сахар для IDisposable, заменяет try-finally.
